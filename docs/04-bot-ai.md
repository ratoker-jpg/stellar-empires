# Bot AI

**Статус:** Draft  
**Последнее обновление:** 2026-07-18

## 1. Цель

Боты должны создавать ощущение живой галактики: развиваться, конкурировать за пространство, ошибаться, адаптироваться и реагировать на игрока.

Бот не должен быть скриптом, который просто строит одинаковую последовательность объектов. Он должен выбирать цели на основании состояния собственной империи и доступной информации о мире.

## 2. Основной принцип

Бот играет через те же команды, что и игрок.

Допустимо:

- анализировать известное состояние;
- планировать несколько действий;
- резервировать ресурсы;
- выбирать цели;
- отправлять игровые команды.

Запрещено:

- напрямую добавлять ресурсы;
- создавать корабли без производства;
- читать скрытые данные игрока без разведки;
- мгновенно перемещать флот;
- игнорировать требования технологий;
- менять мир в обход доменных обработчиков.

## 3. Уровни принятия решений

```text
Стратегия империи
        ↓
План развития
        ↓
План конкретной планеты
        ↓
Оценка угроз и целей
        ↓
Выбор действия
        ↓
Отправка команды
        ↓
Анализ результата
```

### 3.1. Стратегический уровень

Определяет долгосрочное направление:

- экономика;
- технологии;
- экспансия;
- ранняя агрессия;
- оборона;
- смешанная стратегия.

### 3.2. Операционный уровень

Решает:

- какую планету развивать;
- что строить следующим;
- какие исследования нужны;
- сколько ресурсов резервировать;
- какой флот производить.

### 3.3. Тактический уровень

Решает:

- кого разведывать;
- атаковать ли найденную цель;
- каким составом лететь;
- стоит ли отозвать флот;
- куда перебросить ресурсы;
- как реагировать на входящую угрозу.

## 4. Архетипы

| Архетип | Основное поведение | Слабость |
|---|---|---|
| Шахтёр | Максимизирует добычу и склады | Медленно создаёт флот |
| Технолог | Рано открывает продвинутые технологии | Уязвим в начале |
| Экспансионист | Быстро основывает колонии | Распыляет ресурсы |
| Рейдер | Ищет слабые и богатые цели | Зависит от успешных атак |
| Агрессор | Рано производит боевые корабли | Отстаёт в экономике |
| Черепаха | Укрепляет ключевые планеты | Плохо контролирует пространство |
| Логист | Эффективно перераспределяет ресурсы | Слабее в прямом давлении |
| Адаптивный | Меняет приоритеты по ситуации | Требует более сложного планировщика |

Архетип задаёт веса решений, а не жёсткий сценарий.

## 5. Параметры личности

Каждый бот получает значения от 0 до 1:

- агрессивность;
- осторожность;
- склонность к риску;
- терпение;
- жадность;
- мстительность;
- интерес к разведке;
- склонность к экспансии;
- технологический приоритет;
- качество долгосрочного планирования.

Параметры должны заметно влиять на выбор, но не нарушать здравый смысл.

## 6. Память и знания

Бот хранит только доступную ему информацию:

- собственное полное состояние;
- известные координаты;
- разведданные с датой актуальности;
- историю атак;
- оценку силы соседей;
- известные маршруты флотов;
- отношение к другим империям;
- успешность предыдущих решений.

Устаревшая информация теряет точность.

## 7. Оценка действий

Каждое потенциальное действие получает итоговый score.

Пример факторов для здания:

- ожидаемый прирост производства;
- срок окупаемости;
- нехватка энергии;
- требования текущей стратегии;
- влияние на открытие технологии;
- стоимость упущенной возможности;
- риск скорой атаки.

Пример факторов для атаки:

- ожидаемая добыча;
- вероятность победы;
- прогноз потерь;
- время полёта;
- расход топлива;
- вероятность устаревших разведданных;
- риск ответной атаки;
- ценность сохранения флота.

## 8. Планировщик

Бот не должен пересчитывать все варианты каждую секунду.

Планирование запускается:

- после завершения здания;
- после завершения исследования;
- после возвращения флота;
- после получения разведданных;
- после атаки;
- при обнаружении входящей угрозы;
- через редкий контрольный интервал.

Результат планирования — список намерений:

```ts
interface BotIntent {
  priority: number;
  type: string;
  targetId?: string;
  validUntil: number;
  reason: string;
}
```

Перед исполнением намерение повторно проверяется.

## 9. Резервы

Бот должен уметь не тратить всё сразу.

Типы резервов:

- на исследование;
- на колонизатор;
- на аварийную оборону;
- на топливо;
- на грузовые корабли;
- на восстановление после поражения.

Без резервов бот будет постоянно срывать долгосрочные планы мелкими улучшениями.

## 10. Военная оценка

Военная сила не сводится к сумме атаки.

Нужно учитывать:

- состав флота;
- приоритеты целей;
- технологии;
- оборону;
- вероятные подкрепления;
- расстояние;
- ценность цели;
- допустимые потери;
- возможность повторного удара.

Для принятия решения бот может использовать быстрый приблизительный расчёт. Полный бой выполняет общий боевой движок.

## 11. Реакция на угрозу

При входящей атаке бот может:

- вывезти ресурсы;
- отправить флот в безопасную миссию;
- усилить оборону;
- перебросить флот;
- проигнорировать слабую угрозу;
- подготовить ответную атаку.

Решение зависит от времени до прибытия, доступных кораблей и личности бота.

## 12. Сложность

Сложность не должна строиться только на бонусах.

| Уровень | Отличия |
|---|---|
| Лёгкий | Короткий горизонт, слабая оценка целей, больше ошибок |
| Нормальный | Базовое планирование и адекватные резервы |
| Сложный | Длинный горизонт, лучше оценивает риски и логистику |
| Эксперт | Адаптируется, комбинирует стратегии, лучше использует разведданные |

Допустимые экономические коэффициенты должны быть небольшими и явно показанными в настройках партии.

## 13. Производительность

- решения принимаются пакетами;
- простые действия используют эвристики;
- дорогой анализ запускается только при необходимости;
- боты распределяются по временным слотам;
- массовый пересчёт выполняется в Web Worker;
- число рассматриваемых целей ограничивается разумным shortlist.

## 14. Тестирование

Обязательные сценарии:

- бот не застревает без энергии;
- бот достигает первой колонии;
- бот строит флот, соответствующий стратегии;
- бот не атакует заведомо недостижимую цель;
- бот восстанавливается после потери флота;
- разные архетипы дают статистически разные результаты;
- бот не использует скрытую информацию;
- одинаковый seed даёт одинаковое поведение.

## 15. Первый бот

Первый реализуемый профиль — сбалансированный бот:

1. стабилизирует энергию;
2. повышает добычу;
3. открывает разведчика и транспорт;
4. исследует соседние системы;
5. создаёт ограниченный боевой флот;
6. выбирает слабую цель;
7. готовит первую колонию;
8. реагирует на потери пересмотром плана.

До стабильной работы этого профиля новые архетипы не добавляются.
