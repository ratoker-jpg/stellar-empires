### Главный архитектурный вывод

Новые building assets нельзя просто привязать к существующим 12 определениям: пользователь подготовил полный 24-слотовый набор из оригинальной функциональной структуры. Нужен отдельный механический проект расширения каталога, где будет решено:

1. какие 22 обычных функции остаются в PvE-версии;
2. как адаптировать онлайн-зависимые Парламент/Улей/Палату управления и Банк;
3. как работают обелиски и Верховные Галактические Врата в одиночной игре;
4. являются ли науки полностью общими либо имеют общую основу с faction modifiers;
5. как мигрировать существующие save с 12 building IDs на новую схему;
6. какие текущие оригинальные Stellar Empires названия сохраняются, а какие 24 новые asset slugs станут mechanical IDs.

- текущие `aegis` building slugs: `command`, `metal-extractor`, `crystal-refinery`, `gas-extractor`, `power-plant`, `research-lab`, `shipyard`, `sensor-array`, `orbital-depot`, `civic-core`, `tactical-academy`, `defense-foundry`.
- текущие `synod` building slugs: `concord-nexus`, `matter-weave`, `prism-refinery`, `flux-well`, `resonant-core`, `cognition-vault`, `lattice-yard`, `deep-array`, `relay-archive`, `concord-habitat`, `precision-forum`, `shield-foundry`.
- текущие `veyra` building slugs: `swarm-heart`, `alloy-bloom`, `crystal-grove`, `vapor-root`, `solar-membrane`, `memory-pod`, `living-dock`, `pulse-canopy`, `spore-vault`, `nest-cluster`, `hunter-node`, `carapace-forge`.

## 12. Что разрешено использовать дальше

### Можно

- новые сгенерированные building assets из трёх `cleaned-v3` архивов;
- их прозрачные PNG, атласы и манифесты после нормализации схемы;
- этот документ как reference для проектирования оригинальной PvE-механики;
- числовые capture-данные как материал для анализа и последующей ребалансировки, но не как автоматически принятый баланс.

### Нельзя без отдельного разрешения/переработки

- коммитить сохранённые HTML Nemexia, account screenshots и скачанные оригинальные изображения кораблей/обороны/наук;
- подключать оригинальные игровые картинки в runtime;
- копировать тексты описаний дословно; поэтому в этом документе они сведены к функциональным пересказам;
- считать capture-числа готовым балансом Stellar Empires;
- реализовывать неизвестные эффекты обелисков/Врат по догадке.

## 13. Рекомендуемая последовательность следующих PR

1. **Asset intake PR:** добавить три cleaned-v3 архива через CodeX/local import, нормализовать Veyra manifest, зарегистрировать 72 source assets без runtime-подключения.
2. **24-building catalog design PR:** утвердить оригинальные Stellar Empires названия, роли, зоны, ограничения и migration map.
3. **Building runtime PR:** расширить registry, очереди, UI, bots и save migration; подключить новые атласы.
4. **Ship/defense reference adaptation PR:** спроектировать 13/9 каталогов с оригинальными названиями и собственным балансом.
5. **Shared science/commander PR:** определить общие деревья и PvE-адаптацию специальных свойств.
6. **Endgame contract PR:** отдельно определить обелиски, Верховные Врата и победный цикл.

## 14. Acceptance для будущей реализации

- ровно 24 уникальных building definitions и 24 уникальных asset bindings на фракцию;
- отсутствие растягивания кадров: cell 256×256 отображается с сохранением aspect ratio;
- отсутствие повторных кадров между разными building IDs;
- детерминированная migration существующих save;
- одинаковые команды и ограничения для игрока и ботов;
- общие науки и командирские корабли не дублируются по фракционным namespace без причины;
- приоритеты целей и специальные способности покрыты тестами;
- оригинальные Nemexia binaries/screenshots не попадают в source/runtime;
- lint, typecheck, full tests и production build зелёные.

## Итог

Пользовательский пакет действительно содержит полный расширенный reference: **72 здания, 39 обычных кораблей, 13 общих командирских кораблей, 27 средств обороны и 22 общие науки**. Building assets готовы как оригинальные графические исходники, но механическая структура проекта пока вдвое меньше по зданиям и требует отдельного проектирования, а не механической подмены файлов.
