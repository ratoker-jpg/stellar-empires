# Roadmap

**Статус:** Draft  
**Последнее обновление:** 2026-07-18

## Принцип

Каждый этап должен завершаться рабочим, проверяемым состоянием. Новый крупный слой не начинается, пока предыдущий не имеет критериев готовности и автоматических проверок.

## M0 — Документация и границы

### Цель

Зафиксировать, что именно строим и как не раздувать проект.

### Результат

- продуктовая концепция;
- базовые механики;
- архитектура;
- модель ботов;
- модель времени и данных;
- правила разработки;
- ограничения по ассетам и заимствованиям.

### Готово, когда

- документы не противоречат друг другу;
- есть список открытых решений;
- определён первый вертикальный срез.

## M1 — Технический каркас

### Цель

Получить минимальный проект, который стабильно собирается и публикуется.

### Результат

- Phaser 4 + TypeScript + Vite;
- базовая HTML-обвязка;
- линтер, typecheck и тесты;
- GitHub Actions;
- GitHub Pages;
- структура каталогов;
- единая конфигурация игры;
- версия приложения в интерфейсе.

### Готово, когда

- локальная и production-сборки работают;
- pull request запускает проверки;
- merge в `main` обновляет Pages;
- нет ошибок загрузки из подпути `/stellar-empires/`.

## M2 — Мир и сохранение

### Цель

Создать воспроизводимую галактику и надёжное состояние партии.

### Результат

- генерация мира по seed;
- системы, позиции и планеты;
- стартовая империя игрока;
- игровые часы;
- IndexedDB;
- слоты сохранения;
- экспорт и импорт JSON;
- миграция первой схемы.

### Готово, когда

- одинаковый seed создаёт одинаковый мир;
- партия сохраняется и восстанавливается;
- повреждённый импорт не уничтожает рабочий слот;
- состояние не зависит от FPS.

## M3 — Экономический цикл

### Цель

Сделать первую планету функциональной.

### Результат

- металл, кристалл, газ и энергия;
- добыча по времени;
- склады;
- ресурсные здания;
- стоимость и требования;
- очередь строительства;
- отмена строительства;
- отчёты о завершении.

### Готово, когда

- игрок может развить базовую экономику;
- офлайн-пересчёт даёт корректные ресурсы;
- невозможно уйти в отрицательные значения;
- формулы покрыты тестами.

## M4 — Исследования и производство

### Цель

Открыть технологическое развитие и верфь.

### Результат

- дерево исследований;
- общая очередь исследований;
- требования технологий;
- верфь;
- очередь кораблей;
- разведчик, транспорт и первый боевой корабль;
- конфигурация контента через данные.

### Готово, когда

- технологии реально блокируют и открывают контент;
- корабли производятся пакетами;
- очереди сохраняются после перезапуска;
- UI объясняет причину недоступности.

## M5 — Галактика и флоты

### Цель

Дать игроку взаимодействие с внешним миром.

### Результат

- карта галактики;
- выбор систем и планет;
- расчёт расстояния;
- скорость и топливо;
- отправка флота;
- транспорт и передислокация;
- прибытие и возврат;
- отображение активных полётов.

### Готово, когда

- флот нельзя дублировать;
- расход топлива корректен;
- закрытие страницы не ломает полёты;
- одинаковые входные данные дают одинаковое время прибытия.

## M6 — Разведка и бой

### Цель

Завершить первый полноценный стратегический цикл.

### Результат

- разведывательная миссия;
- уровни разведданных;
- оборона;
- боевой движок;
- потери и обломки;
- захват ресурсов;
- боевой отчёт;
- переработчик.

### Готово, когда

- бой воспроизводим по seed;
- отчёт объясняет результат;
- потери корректно применяются к состоянию;
- тесты покрывают ключевые граничные случаи.

## M7 — Первый автономный бот

### Цель

Создать одну бот-империю, способную пройти базовый цикл.

### Результат

- планировщик экономики;
- резервы ресурсов;
- исследовательный план;
- производство флота;
- разведка целей;
- первая атака;
- восстановление после потерь;
- выполнение в Worker.

### Готово, когда

- бот без вмешательства достигает флота и колонии;
- бот не использует скрытые данные;
- бот не застревает в типовых состояниях;
- 100 симуляций завершаются без нарушения инвариантов.

## M8 — Живая галактика

### Цель

Масштабировать симуляцию до нескольких разных империй.

### Результат

- архетипы;
- личности;
- память разведданных;
- отношения и угрозы;
- несколько специализаций планет;
- статистика мира;
- настройка сложности;
- пакетное планирование.

### Готово, когда

- 20–50 ботов развиваются устойчиво;
- архетипы отличаются статистически;
- интерфейс остаётся отзывчивым;
- долгий офлайн-пересчёт не повреждает сохранение.

## M9 — Колонизация и долгосрочная партия

### Цель

Добавить полноценное расширение империй.

### Результат

- лимит колоний;
- колонизатор;
- основание планеты;
- специализация колоний;
- межпланетная логистика;
- борьба за свободные позиции;
- режимы победы.

### Готово, когда

- игрок и боты создают несколько планет;
- экономика масштабируется без ручных исключений;
- условия победы вычисляются корректно.

## M10 — Контент и полировка

### Цель

Расширять игру только после стабильного ядра.

Возможные направления:

- дополнительные фракции;
- больше кораблей;
- командиры;
- редкие объекты;
- дипломатия;
- союзы ботов;
- специальные миссии;
- события галактики;
- улучшенный визуал и звук;
- адаптивный интерфейс.

## Запреты roadmap

До M7 не добавлять:

- сложную дипломатию;
- альянсы;
- героев;
- сюжетные кампании;
- десятки типов кораблей;
- три разные фракции;
- сложную кастомизацию кораблей;
- ручной тактический бой.

Эти системы не улучшают базовый цикл, пока он не работает.

## Первый вертикальный срез

Первая демонстрационная версия должна включать:

> одна планета → добыча → здание → исследование → разведчик → разведка бот-планеты → боевой корабль → атака → отчёт → сохранение.
