# Полный roadmap Stellar Empires

**Статус:** Draft  
**Последнее обновление:** 2026-07-18  
**Основание:** глубокое исследование систем Nemexia, официальной справки, старых релиз-нотов и доступных руководств игроков.

## 1. Назначение roadmap

Этот документ задаёт полный порядок разработки одиночной браузерной космической стратегии с автономными бот-империями.

Проект не является точной копией Nemexia. Мы используем системные идеи жанра, но создаём собственные:

- названия;
- фракции;
- визуальный стиль;
- тексты;
- баланс;
- интерфейс;
- игровые данные;
- программный код.

Главная цель — сохранить стратегическую глубину цикла:

> экономика → исследования → производство → разведка → флот → миссии → бой → расширение → борьба за финальную цель партии.

## 2. Что показало исследование Nemexia

Nemexia состоит не из одной механики строительства, а из нескольких связанных слоёв:

- три асимметричные расы;
- планета с четырьмя функциональными зонами;
- три основных экономических ресурса;
- энергия как ограничивающий ресурс;
- здания, исследования и очереди;
- гражданские, боевые и сервисные корабли;
- около 13 типов кораблей на расу;
- планетарная оборона;
- улучшения кораблей;
- навыки кораблей;
- адмирал и командирские корабли;
- карта галактики, системы, планеты и солнца;
- транспорт, размещение, разведка, атака, колонизация и переработка;
- дополнительные миссии: пиратство, астроэксплуатация, космические путешествия и специальные миссии против нейтральных сил;
- групповой бой с приоритетами целей и ограничением по раундам;
- обломки, захват ресурсов и боевые отчёты;
- альянсы, уровни и способности альянсов;
- общая командная планета;
- солнечные битвы;
- кристаллы, обелиски и финальное сооружение раунда.

Для одиночной игры часть MMO-систем должна быть адаптирована, а не перенесена буквально.

## 3. Принципы реализации

### 3.1. Данные раньше контента

Здания, технологии, корабли, оборона, миссии и фракционные особенности должны описываться данными, а не разрозненными условиями в коде.

### 3.2. Симуляция отдельно от Phaser

Phaser отвечает за:

- карту;
- сцены;
- анимации;
- визуальные эффекты;
- взаимодействие пользователя.

Симулятор отвечает за:

- ресурсы;
- время;
- очереди;
- полёты;
- бои;
- ботов;
- победу и поражение.

### 3.3. Детерминированность

Одинаковый seed и одинаковая последовательность команд должны давать одинаковый результат.

### 3.4. Одни правила для игрока и ботов

Боты не получают ресурсы из воздуха и не читают скрытое состояние игрока. Они используют те же команды, ограничения и формулы.

### 3.5. Событийное время

Мир обновляется через события:

- завершение здания;
- завершение исследования;
- производство корабля;
- прибытие флота;
- начало боя;
- возврат флота;
- решение бота.

### 3.6. Контент добавляется после устойчивой механики

Нельзя начинать три фракции, десятки кораблей и сложную мета-игру до готовности базовой экономики, миссий и боя.

## 4. Контрольные уровни продукта

### Prototype

Работает одна планета, один ресурсный цикл и сохранение.

### MVP

Работает полный базовый цикл:

> экономика → исследование → корабль → разведка → атака → отчёт → восстановление.

### Vertical Slice

Есть несколько планет, колонизация, несколько миссий, рабочий бот и визуально связный интерфейс.

### Alpha

Есть живая галактика с несколькими ботами, фракционные различия и долгосрочная партия.

### Beta

Есть командная мета-цель, полный контент, балансировочные прогоны и стабильные сохранения.

### 1.0

Партия полностью проходима от старта до финальной победы без ручного вмешательства разработчика.

### Цель

Зафиксировать системный состав игры до начала кода.

### Работы

- собрать карту механик Nemexia;
- разделить подтверждённые механики и предположения;
- составить каталог экранов;
- составить каталог игровых сущностей;

- определить системы, адаптируемые под одиночную игру;

- определить неизвестные формулы и данные;
- сформировать таблицу источников;
- принять первоначальные продуктовые решения.

### Обязательные решения

- длительность обычной партии;
- скорость игрового времени;

- документы не противоречат друг другу;
- неизвестные механики отмечены явно;
- определён первый вертикальный срез;
- нет требований «сделать всё сразу».

## M2 — Детерминированное ядро симуляции

### Зависимости

M1.

### Цель

Создать независимый от интерфейса игровой симулятор.

### Работы

- тип `GameState`;
- тип `GameCommand`;
- тип `GameEvent`;
- reducer команд;
- приоритетная очередь событий;
- игровые часы;
- seed-генератор случайных чисел;
- журнал команд;
- журнал событий;
- проверка инвариантов;
- режим ускоренного прогона;
- режим пошагового прогона;
- replay партии из seed и команд.

### Ключевые инварианты

- ресурсы не становятся отрицательными;
- один корабль не находится в двух флотах;
- завершённое событие не выполняется повторно;
- удалённая планета не принимает новые команды;
- время мира не движется назад;
- повторный replay даёт тот же checksum состояния.

### Готово, когда

- симулятор работает без Phaser;
- одинаковый seed даёт одинаковые события;
- 10 000 последовательных событий не нарушают инварианты;
- replay восстанавливает итоговое состояние.

## M12 — Боевой движок v1

### Зависимости

M9, M10, M11.

### Цель

Реализовать воспроизводимый групповой бой.

### Основа, подтверждённая исследованием Nemexia

- корабли и оборона группируются по типам;
- у групп есть приоритетные цели;
- применяются параметры атаки и прочности;
- действуют технологические и иные бонусы;
- бой ограничен числом раундов;
- в Nemexia использовалось до 12 раундов.

# 5. Первый вертикальный срез

Первая демонстрационная версия должна включать:

> одна планета → четыре зоны → добыча → энергия → здание → исследование → верфь → разведчик → разведка бот-планеты → транспорт → боевой корабль → атака → групповой бой → отчёт → обломки → возврат флота → сохранение.

В вертикальный срез не входят:

- три фракции;
- коалиции;
- командная планета;
- адмирал;
- командирские корабли;
- солнечные битвы;
- финальная мегаструктура;
- полный каталог миссий;
- полный набор кораблей.

# 6. MVP

MVP включает:

- одну фракцию;
- одну стартовую планету;
- четыре зоны;
- три экономических ресурса и энергию;
- здания;
- исследования;
- семь базовых ролей кораблей;
- оборону;
- шесть основных миссий;
- групповой бой;
- разведку;
- обломки;
- одного полноценного бота;
- IndexedDB;
- JSON-экспорт;
- GitHub Pages.

# 10. Запреты roadmap

До готовности M14 нельзя добавлять:

- сложные коалиции;
- командную планету;
- финальную мегаструктуру;
- три полноценные фракции;
- полный набор из десятков кораблей;
- сложные командирские способности;
- событийные сезоны;
- сюжетную кампанию;
- ручной тактический бой;
- сетевой мультиплеер.

До готовности M12 нельзя утверждать финальный баланс кораблей.

До готовности M15 нельзя балансировать позднюю мета-игру.

До готовности M23 нельзя считать игру сбалансированной по личным ощущениям.

# 11. Неизвестные данные и исследовательские задачи

Часть точных формул и данных Nemexia недоступна либо менялась между версиями.

Нельзя слепо считать подтверждёнными:

- полные стоимости всех юнитов после ребалансов;
- все коэффициенты атаки и защиты;
- скрытые правила выбора цели;
- точные формулы шпионажа;
- полные параметры адмирала;
- все способности командирских кораблей;
- детали поздних версий событий;
- точный баланс рас версии 5.x.

Для них используется отдельный процесс:

1. зафиксировать источник;
2. отметить версию игры;
3. воспроизвести пример;
4. сравнить с боевым симулятором или логом;
5. признать формулу подтверждённой либо предположительной;
6. спроектировать собственную понятную формулу;
7. проверить её массовыми прогонами.
