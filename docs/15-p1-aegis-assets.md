## Назначение

> добыча → энергия → строительство → исследование → производство корабля → разведка → атака → оборона → отчёт.

## Формат поставки

Ассеты собраны в пять SVG-атласов:

- `buildings-atlas.svg` — сетка 4×2, кадр 256×256;
- `ships-atlas.svg` — сетка 3×2, кадр 512×512;
- `defenses-atlas.svg` — сетка 3×1, кадр 256×256;
- `technologies-atlas.svg` — сетка 3×2, кадр 256×256;
- `effects-atlas.svg` — сетка 3×1, кадр 256×256.

Каждый кадр зарегистрирован в `src/assets/aegisVerticalSliceAssets.ts` с:

- уникальным `asset_id`;
- категорией;
- именем;
- игровой ролью;
- URL атласа;
- точными координатами кадра.

## Правила использования

- UI не должен вручную дублировать координаты кадров;
- код получает их только из manifest;
- игровые сущности связываются с `asset_id`, а не с путём к файлу;
- производные WebP-атласы допускаются после измерения веса и качества;
- SVG остаётся мастер-версией для текущего прототипа;
- новые уровни зданий добавляются отдельным P2-набором.

## Проверка

Автоматический тест проверяет:

- наличие 26 записей;
- совпадение суммы кадров в атласах;
- уникальность идентификаторов;
- принадлежность путей каталогу `aegis/p1`;
- покрытие всех пяти категорий;
- корректные положительные размеры кадров.

## Ограничение

Набор завершает визуальную поставку вертикального среза, но сам по себе не реализует экономику, строительство, исследования или бой. Следующий пакет связывает эти ассеты с настоящими игровыми сущностями и действиями.
