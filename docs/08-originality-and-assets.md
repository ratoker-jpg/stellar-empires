# Originality and Assets

**Статус:** Draft  
**Последнее обновление:** 2026-07-18

## 1. Цель

Проект может вдохновляться жанром и отдельными игровыми принципами классических браузерных стратегий, но должен иметь собственную реализацию, визуальный язык и набор контента.

Этот документ не является юридической консультацией. Он фиксирует внутреннюю безопасную политику проекта.

## 2. Разрешено использовать как вдохновение

- общий жанр космической стратегии;
- развитие планет;
- добычу ресурсов;
- очереди строительства;
- исследования;
- производство флота;
- координатную карту галактики;
- разведку;
- формульные автоматические бои;
- колонизацию;
- отчёты о миссиях;
- автономные империи.

Общие идеи и игровые жанровые паттерны должны быть переработаны под собственные правила проекта.

## 3. Не копируем один в один

- название чужой игры;
- логотип;
- названия рас и персонажей;
- тексты интерфейса и описаний;
- изображения кораблей, планет и зданий;
- музыку и звуковые эффекты;
- анимации;
- конкретный макет экранов;
- исходный код;
- закрытые данные;
- уникальные сюжетные элементы;
- точные наборы характеристик и таблицы баланса, если они являются узнаваемой частью конкретного продукта.

## 4. Собственные элементы Stellar Empires

Проект должен получить:

- собственное название и символику;
- собственную вселенную;
- собственные фракции;
- собственную терминологию;
- собственные корабли и здания;
- собственный интерфейс;
- собственные формулы и баланс;
- собственные визуальные материалы;
- собственное поведение ботов;
- собственные улучшения одиночного режима.

## 5. Временные заглушки

На этапе прототипирования разрешены только нейтральные технические заглушки:

- простые геометрические формы;
- цветовые блоки;
- текстовые иконки;
- самостоятельно созданные изображения;
- ассеты с проверенной лицензией, допускающей нужное использование.

Чужие игровые ассеты не должны попадать в публичный репозиторий даже как временный материал без явного разрешения.

## 6. Реестр ассетов

Для каждого внешнего ассета фиксируются:

| Поле | Описание |
|---|---|
| ID | Внутренний идентификатор |
| Файл | Путь в репозитории |
| Автор | Автор или организация |
| Источник | Где получен |
| Лицензия | Условия использования |
| Изменения | Какие правки внесены |
| Атрибуция | Как требуется указать автора |
| Проверено | Дата проверки |

Реестр будет храниться в `docs/assets-register.md` после появления первых внешних материалов.

## 7. Генеративные материалы

Для изображений и звуков, созданных генеративными инструментами, фиксируются:

- инструмент;
- дата генерации;
- исходное описание;
- использованные референсы;
- ручные изменения;
- итоговый файл.

Не использовать запросы на точное воспроизведение защищённого фирменного стиля конкретного продукта или прямое копирование узнаваемых объектов.

## 8. Данные и механики

Публично доступную информацию можно использовать для исследования жанра, но итоговые игровые таблицы должны быть собраны под собственные цели:

- скорость партии;
- размер мира;
- поведение ботов;
- длительность сессии;
- ускорение времени;
- отсутствие монетизации.

Нельзя строить проект на предположении, что отсутствие коммерции автоматически разрешает копирование материалов.

## 9. Названия

Рабочее название проекта: **Stellar Empires**.

Перед любой публичной публикацией за пределами тестовой GitHub Pages нужно отдельно проверить название на конфликт с существующими играми и товарными знаками.

## 10. Публичный репозиторий

Так как репозиторий публичный:

- любой добавленный файл считается фактически опубликованным;
- нельзя коммитить материалы «только для личного использования»;
- нельзя хранить секреты, токены и приватные файлы;
- спорный ассет должен быть исключён до выяснения прав;
- история Git сохраняет удалённые материалы, поэтому проверка выполняется до commit.

## 11. Правило принятия ассета

Ассет добавляется, только если выполнено одно из условий:

1. Создан специально для проекта и права понятны.
2. Создан владельцем проекта самостоятельно.
3. Имеет явную совместимую лицензию.
4. Получено письменное разрешение правообладателя.
5. Находится в общественном достоянии и это проверено.

Если происхождение неясно, ассет не используется.

## 12. Практическое решение на старт

До утверждения визуального направления используются собственные минималистичные заглушки. Сначала строится рабочая симуляция, затем создаётся оригинальный визуальный набор под уже зафиксированные игровые сущности.
