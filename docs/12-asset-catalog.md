# Asset Catalog

**Статус:** Accepted as production baseline  
**Последнее обновление:** 2026-07-18

## 1. Назначение

Этот документ задаёт полный перечень визуальных и звуковых ассетов Stellar Empires.

Каталог используется для:

- планирования генерации;
- контроля полноты контента;
- единого именования файлов;
- оценки объёма;
- привязки ассетов к этапам roadmap;
- автоматической проверки отсутствующих файлов;
- загрузки ассетов в Phaser;
- отслеживания происхождения и лицензий.

## 2. Статусы

| Статус | Значение |
|---|---|
| `planned` | Нужен, но работа не начата |
| `briefed` | Есть утверждённый бриф |
| `generated` | Есть первый вариант |
| `selected` | Выбран рабочий вариант |
| `processed` | Обработаны фон, размер и формат |
| `integrated` | Подключён в проект |
| `approved` | Проверен в реальном интерфейсе |
| `rework` | Требует переделки |

## 3. Этапы поставки

| Этап | Содержание |
|---|---|
| `P0` | Style prototype: минимальный контрольный набор |
| `P1` | Технический каркас и первая игровая сцена |
| `P2` | Полный вертикальный срез первой фракции |
| `P3` | Три фракции и полная основная игра |
| `P4` | Эндгейм, полировка, аудио и расширенный контент |

## 4. Именование

Формат:

```text
<domain>_<faction>_<type>_<role>_<variant>_<size>.<ext>
```

Примеры:

```text
ship_aegis_cruiser_hero_1024.webp
ship_synod_scout_card_256.webp
building_veyra_hive_zone_l3.webp
icon_shared_resource_metal_32.svg
fx_shared_explosion_large_atlas.webp
```

Коды фракций:

- `aegis` — Директорат «Эгида»;
- `synod` — Машинный Синод;
- `veyra` — Рой Вейра;
- `shared` — общие ассеты.

# 25. Метаданные каждого ассета

Для каждого production-ассета фиксируется:

```ts
interface AssetManifestEntry {
  id: string;
  category: string;
  faction: "shared" | "aegis" | "synod" | "veyra";
  stage: "P0" | "P1" | "P2" | "P3" | "P4";
  status: string;
  sourceType: "generated" | "original" | "licensed";
  sourceNote: string;
  masterPath: string;
  productionPath: string;
  width: number;
  height: number;
  format: string;
  alpha: boolean;
  version: number;
  promptRef?: string;
  checksum?: string;
}
```

# 26. Проверки каталога

Автоматические проверки должны находить:

- отсутствующий файл из manifest;
- файл без записи в manifest;
- дублирующий `asset_id`;
- неверный размер;
- слишком большой вес;
- запрещённый формат;
- отсутствие alpha там, где она обязательна;
- ссылку на ассет другой фракции без явного разрешения;
- отсутствующую заглушку;
- разорванную ссылку в Phaser loader.
